Overview

What is OntoCurator Agent?

In pharmaceutical R&D, scientists write thousands of lab reports, protocols, and assay descriptions every year. Each document is full of biomedical terms — cell types, assays, compounds, processes — many of which may not yet exist in any controlled ontology. OntoCurator Agent automates the most tedious part of ontology governance: finding those terms, mapping them to existing standards, detecting conflicts, and routing decisions to the right stewards.

🔍

Automated Term Discovery

GPT-5.2 reads raw scientific text and identifies every concept that needs ontology coverage — including novel terms that don't exist yet.

Architecture

The 4-Agent LangGraph Pipeline

Each curation run flows through a LangGraph StateGraph with four nodes wired in sequence. Errors at any node are captured in state and surfaced in the final summary without crashing the pipeline.

🔬
Node 1
Extract
EntityExtractorAgent
Sends source text to GPT-5.2 with a NER system prompt. Returns a list of CandidateTerm objects with entity type, confidence, and context. Uses tenacity for 3-attempt retry.
🗺️
Node 2
Map
TermMapperAgent
For each candidate term: searches BioPortal REST API + EBI OLS4, then calls GPT-5.2 to rank results and explain the mapping. Flags terms with needs_new_term=True when no suitable match exists.
⚔️
Node 3
Resolve
ConflictResolverAgent
Passes all mappings to GPT-5.2 and asks it to identify: duplicate, near_synonym, hierarchy_violation, and scope_mismatch conflicts. Returns a structured ConflictReport.
🏛️
Node 4
Govern
GovernanceGateAgent
Rule-based (no LLM). Confidence ≥ 0.88 → auto_approve. 0.60–0.88 → human_review. < 0.60 or new term → escalate. Routes to named domain stewards based on entity type.
State object flowing through the graph
class CurationState(TypedDict): run_id : str input_text : str source_document : Optional[str] candidate_terms : list[dict] # ← filled by Node 1 term_mappings : list[dict] # ← filled by Node 2 conflict_report : Optional[dict] # ← filled by Node 3 curation_results : list[dict] # ← filled by Node 4 errors : list[str] # ← accumulated by any node

Wrote all 4 agent classes, 2 ontology tool clients, and 8 Pydantic schemas. Initialised git repo, made first commit, and pushed to GitHub.
Bug Fix 1 — pip install OSError
Created README.md to satisfy hatchling's readme requirement
pyproject.toml declared readme = "README.md" but the file was missing, causing OSError: Readme file does not exist during pip install -e ".[notebooks]". Created a comprehensive 194-line README.md with architecture docs, usage examples, and governance thresholds. Committed as e8333b2.
Bug Fix 2 — Notebook install typo
Fixed ..[notebooks] → .[notebooks] in notebook cell 3
The install cell used ..[notebooks] which is invalid pip path syntax. Changed to .[notebooks] (with cwd=".." already set, . correctly resolves to the package root).
Bug Fix 3 — GPT-5.2 JSON response format (commit 6eb31c2)
Fixed entity extractor to handle GPT-5.2's single-object json_object response
Discovered that GPT-5.2 returns a bare single-entity JSON object {"text": ..., "entity_type": ...} instead of the expected {"entities": [...]} wrapper when response_format=json_object is used without explicit instruction to use an array wrapper. The parser called parsed.get("entities", []) and got [] — silently producing zero terms.

Fix: Updated the user prompt to explicitly request {"entities": [...]} with an example, and added a three-way parser fallback: {"entities": [...]} → plain list → single-entity dict fallback.

All 12 code cells ran: install → API key load → test data → pipeline init → curation of 3 docs → rich tables → governance charts → confidence histogram → audit trail inspector → review queue → CSV export. Results: 68 terms curated, charts saved, curation_results.csv exported.

Key design decisions, data models, and external integrations.

Data Models (Pydantic v2)

class CandidateTerm(BaseModel): text: str normalized_text: str entity_type: EntityType # enum confidence: float source_document: Optional[str] class TermMapping(BaseModel): candidate: CandidateTerm top_match: Optional[OntologyMatch] mapping_confidence: float needs_new_term: bool mapper_reasoning: Optional[str] class GovernanceDecision(BaseModel): action: GovernanceAction # enum reason: str priority: str assigned_steward: Optional[str]

Governance Thresholds

# GovernanceGateAgent — rule-based, no LLM if mapping_confidence >= 0.88: action = "auto_approve" elif mapping_confidence >= 0.60: action = "human_review" else: # < 0.60 OR needs_new_term action = "escalate" # Domain steward routing table stewards = { "assay": "assay_ontology_steward", "molecule": "cheminformatics_steward", "cell_type": "cell_biology_steward", "gene": "genomics_steward", "disease": "clinical_steward", }

Ontology APIs

🏥

BioPortal REST API

Searches OBI, ChEBI, GO, CL, DOID, EFO, BAO, HP, NCIT, MeSH. Requires BIOPORTAL_API_KEY. Falls back to mock mode (returns empty) when key is absent so the pipeline never crashes.

🔬

EBI OLS4 API

EMBL-EBI Ontology Lookup Service. No authentication required. Scored by match type: exact=1.0, synonym=0.85, substring=0.75.

GPT-5.2 Compatibility Fix

# GPT-5.2 returns a single entity object # instead of {"entities": [...]} wrapper. # Three-way fallback parser: parsed = json.loads(raw) if isinstance(parsed, list): items = parsed # plain array elif "entities" in parsed: items = parsed["entities"] # standard wrapper elif "text" in parsed: items = [parsed] # GPT-5.2 single-obj else: items = []

Get Started

Installation & Usage

1. Clone and install

git clone https://github.com/alishahmohammadi22/onto-curator-agent cd onto-curator-agent # Standard install pip install -e . # With Jupyter notebook extras pip install -e ".[notebooks]"

2. Set API keys

# .env file in the project root OPENAI_API_KEY=sk-... BIOPORTAL_API_KEY=... # optional

3. Run your first curation

from onto_curator import OntoCuratorPipeline pipeline = OntoCuratorPipeline.from_env( model="gpt-5.2", auto_approve_threshold=0.88, human_review_threshold=0.60, ) summary = pipeline.curate( text="Your lab report text here...", source_document="DOC-001" ) print(f"Curated {summary.total_terms} terms") print(f"Auto-curation rate: {summary.auto_curation_rate}%")

4. Explore the notebook

cd notebooks jupyter lab 01_quickstart_demo.ipynb # The notebook covers: # - End-to-end pipeline on 3 documents # - Governance routing charts # - Audit trail inspector # - Human review queue builder # - Confidence histogram # - CSV export
